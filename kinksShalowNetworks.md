# What is the origin of kinks in a shallow neural network?

In a shallow neural network with ReLU activation, **kinks** (breakpoints, or points where the slope changes) arise as a direct consequence of how the hidden units behave.

Each neuron defines a piecewise linear function. The network's output is simply the sum of all of these, so the resulting function is also piecewise linear.

---

## 1. The output of a ReLU neuron

A hidden neuron computes

\[
h_i(x)=\max(0,\theta_i+\omega_i x),
\]

where

- \(\omega_i\) is the input weight,
- \(\theta_i\) is the bias.

The ReLU function has two distinct regimes:

\[
h_i(x)=
\begin{cases}
0, & \theta_i+\omega_i x<0,\\
\theta_i+\omega_i x, & \theta_i+\omega_i x\ge0.
\end{cases}
\]

That is, before a certain point the output stays constant, and after it, the output becomes a linear function.

---

## 2. Where does the kink appear?

The transition occurs when the argument of the ReLU changes sign.

Solving

\[
\theta_i+\omega_i x=0,
\]

we get

\[
x=-\frac{\theta_i}{\omega_i}.
\]

This value is the **kink** or **breakpoint** generated by the neuron.
At this point, the slope of the function changes.

---

## 3. The network's output

If the hidden layer contains three neurons, the network's output is

\[
y=\phi_0+\phi_1h_1+\phi_2h_2+\phi_3h_3,
\]

where

- \(\phi_0\) is the output bias,
- \(\phi_i\) are the weights connecting the hidden layer to the output.

Each term \(\phi_i h_i(x)\) is a piecewise linear function with a single kink.
Summing all these functions gives another piecewise linear function.

---

## 4. Why do several kinks appear?

Each neuron has its own point

\[
x_i=-\frac{\theta_i}{\omega_i}.
\]

When the input value reaches one of these points, a neuron switches state (from inactive to active, or vice versa).

As a result:
- that neuron's contribution changes,
- the overall slope of the output changes,
- a new kink appears in the function represented by the network.

Because of this, a network with several neurons can exhibit multiple slope changes.

---

## 5. Role of the parameters

Each parameter plays a different role:

### Neuron bias (\(\theta_i\))

Horizontally shifts the kink.

\[
x_i=-\frac{\theta_i}{\omega_i}.
\]

Changing the bias moves the point where the slope change occurs.

### Input weight (\(\omega_i\))

Determines two things:
- the position of the kink,
- the slope of the function when the neuron is active.

### Output weight (\(\phi_i\))

Does not change the position of the kink.
It only scales the neuron's contribution, increasing or decreasing the slope change produced in the output.

---

## 6. Geometric interpretation

Each ReLU neuron can be understood as a **hinge function**.

- Before the kink, it contributes a line (or a constant value).
- After the kink, it contributes another line with a different slope.

The neural network builds complex functions simply by summing many of these hinges.

---

## 7. Important consequence

If the input is one-dimensional (\(x\in\mathbb{R}\)) and the network has \(H\) hidden neurons, the resulting function will be piecewise linear and can have, at most, **\(H\) kinks** (assuming they all occur at distinct positions).

This explains why increasing the number of neurons increases the representational capacity of a ReLU network: each neuron adds a new point where the slope can change, allowing increasingly complex functions to be approximated.
